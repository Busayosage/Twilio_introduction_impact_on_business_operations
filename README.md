# Twilio Introduction: Impact on Business Operations

## Project Overview

This project analyses the operational impact of introducing Twilio communication technology within a credit control environment.

The objective of the analysis is to understand whether Twilio helps the business by:

* Reducing unnecessary payment-related calls
* Improving revenue-generating call outcomes
* Releasing operational time for agents
* Improving efficiency in collections operations

The dataset used in this analysis has been anonymised to ensure GDPR compliance while maintaining the analytical structure.

---

## Business Questions (Areas Considered)

The analysis focuses on three key operational questions.

### 1. Reduction in Payment Calls

Agents often spend significant time handling administrative calls such as:

* Balance enquiries
* Stop removed requests
* Payment confirmations

If these calls decrease after the introduction of Twilio, agents can focus on higher-value collections activity.

---

### 2. Do Any Call Outcomes Increase Revenue?

Not all calls generate business value.

Revenue-linked outcomes include:

* Payment Taken
* Promise to Pay
* Account Resolution

This project investigates whether the introduction of Twilio increases these outcomes.

---

### 3. How Much Operational Time Can Be Recovered?

Operational time saved through improved communication workflows can be redirected toward productive collections activity.

The analysis evaluates:

* Total call duration
* Agent workload distribution
* Potential time savings

---

## Analytical Process

The analysis follows a structured pipeline:

Raw Data
→ Data Cleaning
→ Pivot Table Analysis
→ KPI Metrics
→ Operational Dashboard

---

## Dashboard

The dashboard summarises key operational metrics including:

* First Time Resolution proxy
* Payment call rate
* Revenue outcome rate
* Avoidable call rate
* Agent call distribution
* Call status breakdown

---

## Tools Used

* Microsoft Excel
* Pivot Tables
* Business KPI Analysis
* Operational Dashboard Design

---

## Data Privacy

To comply with GDPR and internal data protection requirements:

* Customer identifiers were replaced with dummy IDs
* Agent names were anonymised
* Phone numbers were removed
* Dates were shifted to a synthetic timeline

This preserves the analytical logic while protecting sensitive information.
